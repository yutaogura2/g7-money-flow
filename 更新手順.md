# 更新手順（Claude向け運用メモ）

## フロー
1. 対象公式ソースを巡回し、新規/未登録の声明・コミュニケを探す。
2. 各件について以下を作成し `data.json` の `meetings[]` に追記:
   - 日付 / 種別 / 議長国 / 原題 / 邦題 / 原文URL / 原語
   - 要約(和) / 決定事項(和) / 主要部分の全訳(和)
   - 考察4観点（為替金利・商品・セクター・地政学）/ 方向性メモ / タグ
3. 関連イベントを `calendar[]` に追記（G7＋G20/FOMC/日銀/ECB等）。状態は予定/済。
4. `meta.last_updated` を更新。
5. `python build.py` を実行 → `data.js` と `G7_一次情報ログ.xlsx` を再生成。
6. 出典が確認できない情報は記載しない（推測で埋めない）。

## meetings[] のスキーマ
```jsonc
{
  "id": "2025-06-kananaskis-summit",   // 一意なID（年月-会場-種別など）
  "date": "2025-06-17",
  "type": "首脳会議",                   // 首脳会議 / 財務相・中銀総裁会議 / 外相会合 / シェルパ会合 等
  "presidency": "カナダ",
  "title_orig": "原題（原語のまま）",
  "title_ja": "邦題",
  "source_url": "https://...",          // 公式の一次情報URL
  "source_lang": "en",
  "summary_ja": "要約",
  "key_points_ja": ["決定事項1", "決定事項2"],
  "full_translation_ja": "主要部分の全訳（改行は \\n）",
  "analysis": {
    "fx_rates": "為替・金利への含意",
    "commodities": "商品・エネルギー・金への含意",
    "sectors": "セクターへの含意",
    "geopolitics": "地政学・リスク全般"
  },
  "market_direction": "方向性メモ",
  "tags": ["タグ1", "タグ2"]
}
```

## calendar[] のスキーマ
```jsonc
{
  "date": "2026-06-15",
  "name_ja": "イベント名",
  "category": "G7首脳",                 // G7首脳 / G7財務 / G20 / FOMC / 日銀 / ECB / 経済指標 等
  "status": "予定",                     // 予定 / 済
  "related_meeting_id": null,            // 対応するmeetings.idがあれば
  "note": "備考"
}
```

## 対象公式ソース（一次情報）
- G7議長国サイト（年ごとに変わる。例: 2025カナダ、2026フランス）
- 各国財務省: 米国財務省 home.treasury.gov / 英 HM Treasury gov.uk / 日本 財務省 mof.go.jp・外務省 mofa.go.jp / 独・仏・伊・加 財務省
- EU: 欧州理事会 consilium.europa.eu / 欧州委員会 ec.europa.eu
- 中央銀行: FRB federalreserve.gov / 日銀 boj.or.jp / ECB ecb.europa.eu

## マクロ数値データ（週1回）
1. 初回のみ: FRED無料APIキーを取得し `fred_api_key.txt` に1行で保存
   （https://fred.stlouisfed.org/docs/api/api_key.html）。このファイルはコミットしない。
2. `python fetch_macro.py` … FREDから `macro_data/*.csv` を更新（未取得シリーズは標準エラーに表示）。
3. `data.json` の `macro_analysis`（global_liquidity / fx_rates / implications / updated）を最新の数値を踏まえて記述。
4. `python build.py` … `data.js` と Excel を再生成。
5. `data.json` と `macro_data/` をGitコミット（履歴保存）。`fred_api_key.txt` はコミットしない。

### 対象シリーズの追加・修正
`series_config.json` に `{id, country, indicator, unit, transform}` を追加。
`transform` は `level`（原系列）または `yoy_pct_also`（前年比%も併記）。
`source` を省略するとFRED、`"ecb"`/`"boj"` で各公式ソースから取得。
FRED等に存在しないidは取得時にスキップ＆ログ表示されるので、ログを見て修正する。

### データソース別メモ
- 米国・他: FRED（要APIキー）。
- ユーロ圏M2/M3: ECB公式API（`source:"ecb"`、`ecb_key`指定。無料・キー不要）。
- 日本M2: 日銀 主要時系列フラットCSV `md02_m_1.csv`（cp932）から自動取得（`source:"boj"`）。
  M2平残をデータコード `MD02'MAM1NAM2M2MO` で特定し億円→兆円換算。週次タスクで自動更新。

## 自動更新（2系統）
### ① クラウド（GitHub Actions）= データの正本
`.github/workflows/update-data.yml` が**毎週日曜22:00 UTC（月曜7時JST）**に
クラウド上でFRED/ECB/日銀/CFTCを取得→build→push。GitHub Pages（公開サイト）が自動で最新化される。
- FRED APIキーはリポジトリのSecret `FRED_API_KEY`。手動実行: GitHubのActionsタブ、または `gh workflow run update-data.yml`。
- 公開サイト: https://yutaogura2.github.io/g7-money-flow/

### ② ローカル（Windowsタスク）= ローカルExcel/表示の同期のみ
`update_macro.bat` は **`git pull` → `python build.py`** だけ（取得・commit・pushはしない）。
クラウド最新を取り込み、ローカルの `G7_一次情報ログ.xlsx`・`data.js` を再生成する。CIと履歴が分岐しない。
- タスク名: `G7_MacroWeeklyUpdate`（毎週月曜 09:00、PCログオン時に実行）
- ログ: `update_macro.log`（Git除外）
- 手動実行: `update_macro.bat` をダブルクリック
- 確認: PowerShellで `Get-ScheduledTaskInfo -TaskName G7_MacroWeeklyUpdate`
- 一時停止: `Disable-ScheduledTask -TaskName G7_MacroWeeklyUpdate`
- 削除: `Unregister-ScheduledTask -TaskName G7_MacroWeeklyUpdate -Confirm:$false`
- 時刻変更や考察(`macro_analysis`)の更新は私（Claude）に依頼。
  自動更新は数値データのみ。考察ナラティブはセッションで私が更新する。
- 週次ブリーフィング: build時に `週次ブリーフィング.md` を自動生成（リスクレジーム・主な変化・極端な水準・ポジション偏り・データ鮮度）。概要タブにも表示。

## 主要国財務省の政策（質的情報）
G7声明と同様、`data.json` の `meetings[]` に **種別「財務省政策」** で追記する
（例: 日本MOFの為替介入、米財務省の国債発行計画、中国の財政刺激）。既存の会議ログ・
考察ビュー・Excelにそのまま反映される（コード変更不要）。
