## 1. 目的

プロのトレーダーが常用する市場・リスク指標（金利スプレッド・クレジット・ボラ・為替・期待インフレ・実質金利・金融環境・高頻度景気・原油・**FRBネット流動性**）を追加し、新タブ「市場・リスク」に集約する。全てFRED（および合成計算）で現行値が取得できることを実証済み。

## 2. スコープ（確定要件）

- 追加指標（A）= FRED 14系列（下表）。`group:"market"`、`transform:"level"`。

- 配置 = 新タブ「市場・リスク」。既存「マクロ」は `group` 非market（fundamental）のみ表示に変更。
- 既存 fred/ecb/boj/manual・build・Excel・週次自動更新は流用（小変更のみ）。

## 3. アーキテクチャ

```
series_config.json (group: fundamental|market, source: fred|ecb|boj|computed)
        │
   run() → get_rows(source分岐)
        ├─ fred/ecb/boj/manual（既存）
        └─ computed → fetch_netliq(api_key): WALCL/WTREGEN/RRPONTSYD をFRED取得→整列・換算→[(date,兆ドル)]
        │
   compute_yoy → write_series_csv（既存）
        │
   build.py（payloadに group を付与）→ data.js / Excel
        │
   index.html: renderMacro=fundamental, renderMarket=market（共通描画関数をDRYに）
```

## 4. fetch_macro.py の変更

- 定数: `NETLIQ_COMPONENTS = {"walcl":"WALCL","tga":"WTREGEN","rrp":"RRPONTSYD"}`。
- `_nearest_prior(sorted_pairs, target_date) -> float|None`: 日付昇順の `[(date,val)]` から `date<=target` の最新値を返す。
- `compute_netliq(walcl, tga, rrp) -> list[tuple[str,float]]`:
  - 入力は各 `[(date,val)]`（parse済み）。WALCLの各日付について、TGAは同手法で `date<=` 最新、RRPも `date<=` 最新を取り、
  - 兆ドル = `(walcl_百万 − tga_百万 − rrp_十億×1000) / 1e6`。日付昇順で返す。
- `fetch_netliq(api_key, fred_fetcher=fetch_series) -> list`: 3系列を取得→`parse_observations`→`compute_netliq`。
- `get_rows(...)` に `source=="computed"` 分岐を追加 → `computed_fetcher(series, api_key)`（既定は `compute=="netliq_us"` → `fetch_netliq`）。
- `run(...)` に `computed_fetcher=None`（既定）を追加し get_rows に渡す。既存ソースは不変。

## 5. build.py の変更
- `build_macro_payload` の各系列dictに `"group": s.get("group", "fundamental")` を追加（他は不変）。
- `macro_timeseries_rows`・Excel は全系列を従来どおり出力（市場系も `マクロ_最新`/`マクロ_時系列` に入る。分類列は追加しない）。

## 6. index.html の変更
- 既存 `renderMacro` の本体を共通関数 `renderIndicatorGrid(viewId, seriesList, selectId)` に切り出し（テーマ別グルーピング＋カテゴリフィルタ。既存挙動を維持）。
- `renderMacro()` = `renderIndicatorGrid('view-macro', macro.filter(s=>s.group!=='market'), 'macro-ind')`。
- 新規 `renderMarket()` = `renderIndicatorGrid('view-market', macro.filter(s=>s.group==='market'), 'market-ind')`。
- nav に `<button data-view="market">市場・リスク</button>`、`<section id="view-market">` を追加。`switchView` の配列に `'market'` 追加。`init()` で `renderMarket()` を呼ぶ。
- 既存のチャート/レンジキャプション/カードはそのまま流用。

## 7. エラーハンドリング・正確性
- FRED各系列・ネット流動性の取得失敗はスキップしログ（既存方針）。`level` 系列は前年比「—」表示。
- 単位換算は明示（百万/十億の混在を吸収）。値はFRED公式のみ。
- ネット流動性は週次（WALCL基準）で算出。RRP/TGAは直近営業日値を採用。

## 8. テスト
- `compute_netliq`: サンプル3系列（単位混在・頻度差）を入力に、整列・換算・兆ドル算出が正しいこと（例: WALCL 6,736,424百万・TGA 880,713百万・RRP 3.925十億 → ≈5.8518兆ドル）を検証。
- `_nearest_prior`: 直近以前の値を返すこと（境界・欠如）を検証。
- `run`/`get_rows`: `source:"computed"`/`compute:"netliq_us"` で `fetch_netliq` が呼ばれ、CSVが書かれること（フェッチャ注入）。
- build payload に `group` が含まれること。既存テストを壊さない（`python -m pytest` 全PASS）。
