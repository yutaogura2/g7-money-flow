## 1. 目的

## 3. week.csv の解析仕様

- 列: 0=期間、対外（1-3株式取得/処分/net, 4-6中長期債, 7小計net, 8-10短期債, **11=対外合計net**）、対内（**12-14株式**(14=net), 15-17中長期債, 18小計net, 19-21短期債, **22=対内合計net**）。単位: 億円。
- 期間形式: `2026．6．21～6．27`（全角`．`/`～`、空白混入あり）。正規化（`．`→`.`、`～`→`~`、空白除去）後、`^(\d{4})\.(\d{1,2})\.(\d{1,2})~(\d{1,2})\.(\d{1,2})$` で解析。**期間終了日を系列日付**とし、終了月<開始月なら年+1（年跨ぎ）。
- 値: カンマ・空白除去して float。**億円→兆円（÷10000、4桁丸め）**。
- 抽出3系列: `flow_out`=列11、`flow_in`=列22、`flow_in_eq`=列14。

## 4. 介入CSVの解析仕様

- データ行: 列3=Year（空なら前行から繰越し）、列4=英語月名（空なら繰越し）、列5=Day、列6=金額(億円)。**列5と列6が数値の行のみ**が日次介入（四半期合計行は列5が空）。
- 月次集計: `(YYYY-MM-01, 月内合計÷10000 兆円)`。**介入実施月のみ**（ゼロ月は含めない）。
- 表示名に「実施月のみ」を明記。この系列は鮮度警告の対象外とする（下記 `no_stale`）。

## 5. fetch_macro.py の変更

- 定数: `MOF_WEEK_CSV` / `MOF_FEIO_CSV`（上記URL）。
- モジュール内キャッシュ `_MOF_CACHE`（URL→デコード済みテキスト）で week.csv の3重取得を回避。
- `_parse_mof_week_csv(text, kind) -> list[(YYYY-MM-DD, 兆円)]`（kind: flow_out/flow_in/flow_in_eq）
- `fetch_mof_flows(kind, urlopen=...) -> list`（取得＋cp932デコード＋キャッシュ＋解析）
- `_parse_mof_intervention(text) -> list[(YYYY-MM-01, 兆円)]`、`fetch_mof_intervention(urlopen=...)`
- `mof_dispatch(series, urlopen=...)`: `series["mof_kind"]` で分岐（`intervention`→介入、他→flows）。
- `get_rows(...)` に `source=="mof"` 分岐（`mof_fetcher(series)`、既定 `mof_dispatch`）。`run(...)` に `mof_fetcher=None` を追加。

## 6. build.py の変更（no_stale）

- `series_config` の `"no_stale": true` を持つ系列は `stale=False, stale_days=0` とする（介入系列は「無い月がある」のが正常のため）。
