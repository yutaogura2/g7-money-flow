## 1. 目的・背景

「マネーサプライ(M2)」に**日本**を加える。日銀のマネーストックは、stat-searchの**主要時系列フラットCSV**が固定URLで取得でき、FRED/ECBと同様に自動取得できることを実証した。

- 取得元（実証済み）: `https://www.stat-search.boj.or.jp/ssi/mtshtml/csv/md02_m_1.csv`（**cp932エンコード**）。
- M2 平均残高（億円）＝データコード `MD02'MAM1NAM2M2MO`（CSVの該当列）。最新 2026/05 = 12,980,932 億円（≈1,298兆円）。
- 既存の multi-source 基盤（`source: fred|ecb`）に **`boj`** を追加するだけで統合できる。

## 2. スコープ（確定要件）

| 項目 | 決定 |
|---|---|
| 追加 | 日本M2（`source: "boj"`、自動取得） |
| 取得元 | 日銀 主要時系列CSV `md02_m_1.csv`（cp932）、M2平残列 |
| 単位 | 兆円（億円→兆円に換算: 値/10000） |
| 前年比 | 既存 `compute_yoy` で水準から算出（他系列と同一） |
| 自動化 | 週次無人タスクに乗る（FRED/ECBと同じ） |

### スコープ外（YAGNI）
- M3/マネタリーベース等の追加、BOJ CSVの他系列の網羅。

## 3. アーキテクチャ（multi-sourceに boj を追加）

```
series_config.json (source: "boj")
        │ run() → get_rows(source分岐) → boj_fetcher()
        ├─ fred / ecb（既存）
        └─ boj : fetch_boj_m2() ── md02_m_1.csv 取得(cp932) → M2平残列を抽出 → 兆円換算 → [(YYYY-MM-01, 兆円)]
        │
   compute_yoy → write_series_csv（既存）→ build.py / index.html（無変更）
```

各ソースは共通行形式 `[(YYYY-MM-01, float)]` を返すため、YoY計算・CSV書出・build・表示は無変更で流用。

## 4. fetch_macro.py の変更

- `_parse_boj_m2_csv(text: str) -> list[tuple[str,float]]`:
  cp932デコード済みCSVを解析。「データコード」行から `MD02'MAM1NAM2M2MO`（M2平残）の列を特定し、`YYYY/MM` 形式のデータ行から値を取り、**億円→兆円（/10000）**、日付を `YYYY-MM-01` に正規化、欠損除外、日付昇順で返す。
- `fetch_boj_m2(url=BOJ_M2_CSV, urlopen=...) -> list`:
  CSVをバイト取得 → `cp932` でデコード → `_parse_boj_m2_csv`。
- `get_rows(...)` に `boj_fetcher=None` を追加し、`source=="boj"` で `boj_fetcher()` を呼ぶ。
- `run(...)` に `boj_fetcher=None`（既定 `fetch_boj_m2`）を追加し、get_rows に渡す。`source=="manual"` のスキップ機能（実装済み）はそのまま温存。
- 定数 `BOJ_M2_CSV = "https://www.stat-search.boj.or.jp/ssi/mtshtml/csv/md02_m_1.csv"`。

## 5. 設定（series_config.json）

```json
{ "id": "JP_M2_BOJ", "country": "日本", "indicator": "マネーサプライ(M2)",
  "unit": "兆円", "transform": "yoy_pct_also", "source": "boj" }
```

CSVファイル名は `JP_M2_BOJ.csv`。

## 7. エラーハンドリング・正確性
- BOJ取得/解析失敗はそのシリーズをスキップしログ（既存方針）。FRED/ECBは影響を受けない。
- cp932デコード固定。データコードで列を特定（列順変更に頑健）。値は日銀公式CSVのみ。
- 念のため、`compute_yoy`で算出した最新YoYが日銀公表YoY（2026/05は+2.5%付近）と整合することを実装時に目視確認。

## 8. テスト
- `_parse_boj_m2_csv`: 「データコード」行・単位行・複数月のサンプルCSV（M2平残列を含む）を入力に、`[(YYYY-MM-01, 兆円換算値)]` を日付昇順で返すこと（列特定・億円→兆円・日付正規化・欠損除外）を検証。
- `get_rows`/`run`: `source:"boj"` で boj_fetcher が呼ばれ、CSVが書かれること（fetcher注入）。既存 fred/ecb/manual テストを壊さない。
