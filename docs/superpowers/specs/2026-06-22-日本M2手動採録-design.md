## 1. 目的・背景

「マネーサプライ(M2)」を米国・ユーロ圏に加え**日本**も並べたい。ただし日銀のマネーストックは**無料の安定API/CSVが無い**（stat-searchはJS/CGI駆動、一括DL対象外、e-Statにも無い）。脆いスクレイパは正確性を損なうため採らない。

→ **手動採録ソース**として実装する：日銀公式の月次マネーストック公表値を私（Claude）が採録し、他系列と同じCSV形式で蓄積する。`fetch_macro.py` は手動ソースを自動取得対象から除外し、手動CSVを上書きしない。

> 補足: 日本の流動性は既に「日銀バランスシート（FRED:JPNASSETS・現行）」でカバー済み。本件はM2を明示的に加える補完。

## 2. スコープ（確定要件）

| 項目 | 決定 |
|---|---|
| 追加 | 日本M2（`source: "manual"`） |
| データ源 | 日銀 公式マネーストック公表（月次）。Claudeが `date,value,yoy_pct` を採録 |
| 単位 | 兆円（日銀公表に合わせる） |
| 初期投入 | 公表表にある直近1年程度。以降は毎月1行追記 |
| 自動化 | 週次無人タスクの対象外（手動採録は更新時にClaudeが実施）。タスクは手動CSVを壊さない |

### スコープ外（YAGNI）
- 日銀CGIの自動スクレイピング、他の日本統計、M3/マネタリーベース等。

## 3. アーキテクチャ（既存枠組みの最小拡張）

```
series_config.json (source: "manual")
        │  fetch_macro.run(): manualはスキップ（取得・上書きしない）
macro_data/JP_M2_MANUAL.csv ← Claudeが日銀公表値を採録（date,value,yoy_pct）
        │
build.py（無変更・ソース非依存でCSVを読む）
        ├─→ data.js（マクロ/マネーの流れに反映）
        └─→ Excel（マクロ_最新/時系列に反映）
```

- 既存の `_read_series_csv`/`build_macro_payload`/`macro_timeseries_rows` はソースを問わず `macro_data/<id>.csv` を読むため、**build.py・index.htmlは無変更**で日本M2が表示される（「マネーサプライ(M2)」カテゴリに日本が加わり、マネーの流れのM2比較にも日本が入る）。

## 4. fetch_macro.py の変更

- `run()` のループ先頭で `s.get("source") == "manual"` の系列は **`continue`（スキップ）**。取得もCSV書き込みもしない＝手動CSVを保全。
- 戻り値に `"skipped"`（manual系列のidリスト）を追加し、`main()` で「手動（スキップ）」と表示。
- 既存の fred/ecb 経路・後方互換は不変。

## 5. 設定追加（series_config.json）

```json
{ "id": "JP_M2_MANUAL", "country": "日本", "indicator": "マネーサプライ(M2)",
  "unit": "兆円", "transform": "yoy_pct_also", "source": "manual" }
```

`transform` は表示ラベル整合のため付すが、手動のためfetchでは未使用。YoYは採録時にCSVへ直接記入する。

## 6. データファイル（macro_data/JP_M2_MANUAL.csv）

- 形式: `date,value,yoy_pct`（他のyoy系列と同一）。`date` は `YYYY-MM-01`。
- 値: 日銀公式マネーストックの **M2 平均残高（兆円）** と **前年比%**。
- 初期: 公表表の直近約12か月を投入。`build.py`の履歴/前年比表示が成立する。
- 更新: 毎月、最新月を1行追記（Claudeが公表値を採録）。

## 7. ドキュメント更新（更新手順.md）

「日本M2は手動採録」を明記:
- 手順: 日銀マネーストック公表（月次）を確認 → `macro_data/JP_M2_MANUAL.csv` に最新月を `date,value,yoy_pct` で追記 → `python build.py` → コミット。
- 注意: 数値は必ず日銀公式公表値を正とする（推測しない）。出典を残す。

## 8. エラーハンドリング・正確性
- 手動CSVが未整備でも build は壊れない（ファイルが無ければ単にスキップ、既存仕様）。
- fetch_macro は manual を触らないため、自動更新で手動値が消える事故が起きない。
- 値は日銀公式公表のみを採録（二次情報の数値は使わない）。
