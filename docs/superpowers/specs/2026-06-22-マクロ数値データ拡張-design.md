## 1. 目的

既存のG7一次情報ツールに、**世界の中央銀行のマネーサプライ等のマクロ数値データ**を加え、**主要国財務省の政策**も踏まえて、**世界のマネーの流れ**を投資判断のために把握する。数値は正確に蓄積し、履歴を残して分析できるようにする。

- 量的データ（マネーサプライ・中銀バランスシート・金利・物価・景気雇用）は **FRED（公式統計集約）からAPIで機械取得** し、転記ミスのない正確な時系列として蓄積する。
- 質的情報（G7声明・**主要国財務省の政策**）は既存の「Claudeが公式一次情報を巡回・翻訳・考察」方式を踏襲する。
- 既存のダッシュボード・Excel・Git履歴に統合する。

> 考察は一次情報・公式統計に基づく参考情報であり、投資助言・売買推奨ではない。

## 3. 採用アーキテクチャ（A案：FRED＋CSVストア＋インラインSVGチャート）

```
FRED API ──(fetch_macro.py, 週1)──▶ macro_data/<series>.csv  （正確な全履歴・Git追跡）
                                            │
G7声明・財務省政策(data.json) ──┐          │
                                ▼          ▼
                            build.py（CSV＋data.json を統合）
                                ├─▶ data.js（ダッシュボード：チャート＋最新値）

```

- 数値の真実の源（single source of truth）は `macro_data/*.csv`（シリーズ別の時系列）。質的情報は引き続き `data.json`。
- ダッシュボードのチャートは**依存ライブラリなしのインラインSVG折れ線**で描画（`file://` でオフライン動作を維持）。
- FRED取得は Python 標準ライブラリ（urllib）で行い、**新規依存を増やさない**（openpyxl は既存）。

### 採用理由（他案との比較）
- **B案（単一JSON＋Chart.js）**：チャートは綺麗だがJSライブラリ同梱、大きなデータでGit差分が読みにくい。→ 不採用。
- **C案（SQLite＋pandas）**：本格DBだがバイナリでGit差分が無意味、依存が重く個人用途にオーバースペック。→ 不採用。

## 5. データ取得（FRED）

- 無料APIキー: https://fred.stlouisfed.org/docs/api/api_key.html （登録で即発行）。`fred_api_key.txt` に1行で保存。
- 取得エンドポイント: `https://api.stlouisfed.org/fred/series/observations?series_id=<ID>&api_key=<KEY>&file_type=json`（標準ライブラリ urllib で取得）。
- `series_config.json` 例（series_id は実装時に各々FREDで存在確認する）:

```jsonc
{
  "series": [
    { "id": "M2SL",     "country": "米国",     "indicator": "マネーサプライ(M2)", "unit": "10億ドル",   "transform": "yoy_pct_also" },
    { "id": "WALCL",    "country": "米国",     "indicator": "中銀バランスシート",  "unit": "百万ドル",   "transform": "level" },
    { "id": "FEDFUNDS", "country": "米国",     "indicator": "政策金利",            "unit": "%",          "transform": "level" },
    { "id": "DGS10",    "country": "米国",     "indicator": "10年国債利回り",      "unit": "%",          "transform": "level" },
    { "id": "CPIAUCSL", "country": "米国",     "indicator": "CPI",                "unit": "指数",       "transform": "yoy_pct_also" }
    // … ユーロ圏・日・英・独・仏・伊・加・中国の各指標を同形式で定義
  ]
}
```

- `transform`: `level`=原系列、`yoy_pct_also`=原系列に加え前年同月比%も算出してCSVに併記。
- CSV形式: ヘッダ `date,value`（`yoy_pct_also` の場合は `date,value,yoy_pct`）。UTF-8。
- 取得不能・欠損のシリーズは**スキップして「データなし」と明示**（推測で埋めない）。実行ログに未取得シリーズを出力。

## 6. ダッシュボード追加ビュー（index.html）

- **マクロ**: 国×指標ごとのカード。最新値・前回比/前年比・直近の小さな折れ線グラフ（インラインSVG）。指標別/国別でフィルタ。
- **マネーの流れ**（目玉）:
  - 主要国（米・ユーロ圏・日・中）の **M2前年比** と **中銀バランスシート** を横並び比較（複数系列の折れ線）。
  - 政策金利・10年利回りの一覧表。
  - 数値サマリ（最新値・前年比・方向）は **build.py が自動計算**。
  - 「世界の流動性は拡大/縮小局面か」「為替・株・金への含意」の **考察ナラティブは私が週次更新で記述**し、`data.json` に新設する `macro_analysis` フィールド（`{ "updated": "YYYY-MM-DD", "global_liquidity": "...", "fx_rates": "...", "implications": "..." }`）に保持する。質的情報は `data.json`、機械設定は `series_config.json` と役割を分離する。

チャートは依存なしのSVG（折れ線・軸・凡例を最小限）。日本語UI、`file://` で動作。

## 7. Excel追加シート（G7_一次情報ログ.xlsx）

- `マクロ_最新`: 国×指標の最新スナップショット（国・指標・最新値・日付・前年比）。
- `マクロ_時系列`: ロング形式（`日付・国・指標・値・前年比`）。蓄積用。既定で **2000年以降**（`series_config.json` の `history_start` で変更可）。
- 既存シート（会議ログ・カレンダー・考察サマリ）は維持。`build.py` 実行で毎回再生成。

## 8. 更新フロー（週1回）

1. `python fetch_macro.py` … FRED取得 → `macro_data/*.csv` 更新（未取得シリーズはログ表示）。
2. 私がG7声明・**主要国財務省の政策**の新規を `data.json` に追記し、「マネーの流れ」考察を更新。
3. `python build.py` … `data.js` と `G7_一次情報ログ.xlsx` を再生成。
4. `data.json` と `macro_data/` をGitコミット（履歴保存）。`fred_api_key.txt` はコミットしない。

## 9. エラーハンドリング・正確性

- FRED取得失敗（ネットワーク/不正キー/未知のseries_id）はそのシリーズをスキップし、標準エラーに明示。全体は止めない。
- 数値はFREDから機械取得＝転記ミスなし。CSVをGit追跡し、更新差分で「いつ何が変わったか」を完全保持。
- APIキーは `fred_api_key.txt`（gitignore）に分離し、コードやコミットに含めない。
- 取れない/未確認のシリーズは「データなし」と表示し、推測値を入れない。
