# 政策→フロー 第1弾（グローバル流動性・金利差・政策文脈）設計書

- 作成日: 2026-06-26
- 対象: 既存「G7一次情報＋マネーの流れ ダッシュボード」への拡張
- 位置づけ: 「各国の政策から資金の流れを掴む」強化の **第1弾（背骨）**。第2弾=財務省フローデータ、第3弾=政策ログ/マーカー/統合ビュー は別サイクル。

## 1. 目的

政策→資金の流れの「背骨」を作る。①世界のマネー総量を1本線で（グローバル流動性・ドル換算）、②為替の源泉（日米・米欧金利差）を常時監視、③週次ブリーフィングに今後の政策予定を組み込む。**すべて既存データ源（FRED）と既存インフラで実装**（新規外部依存なし）。

## 2. スコープ（確定要件）

- B1 グローバル流動性指数（ドル換算）: computed 1系列＋マネーの流れビュー先頭に大チャート。
- B2 金利差: 汎用 `spread` computed ＋ 初期3系列（日米10年・米欧10年・米欧政策）。
- D2 ブリーフィング政策文脈: calendar から今後14日の予定行を追加。
- 週次CI・Pages・Excelは無変更で自動反映。

### スコープ外（YAGNI）
- 財務省フローデータ（第2弾）、政策サイクル時計・イベントマーカー・統合ビュー（第3弾）、BoE/加中銀のBS追加。

## 3. B1 グローバル流動性指数（ドル換算）

- 検算済み（2026-06 実データ）: **合計 ≈ 17.87兆ドル**（Fed 6.74 / ECB 7.02×EURUSD / BOJ 4.12÷USDJPY）。
- `fetch_global_liq(api_key, fred_fetcher=fetch_series) -> list`:
  - FRED 5系列: `WALCL`(百万ドル・週次) / `ECBASSETSW`(百万ユーロ・週次) / `JPNASSETS`(億円・月次) / `DEXUSEU`(EURUSD・日次) / `DEXJPUS`(USDJPY・日次)。
  - **WALCLの日付に整列**（他は `_nearest_prior`。いずれか欠けはスキップ、USDJPY=0除外）。
  - `兆ドル = (WALCL + ECBASSETSW×EURUSD + JPNASSETS×100÷USDJPY) / 1e6`、4桁丸め。
- `computed_dispatch` に `global_liq` を追加。
- 設定: `{ "id":"GLOBAL_LIQ_USD", "country":"世界流動性(ドル換算)", "indicator":"流動性", "unit":"兆ドル", "transform":"yoy_pct_also", "source":"computed", "compute":"global_liq", "group":"market", "risk_dir":"low" }`
- 表示:
  - 「市場・リスク」の「流動性」テーマに自動でカード追加（既存機構）。
  - **「マネーの流れ」ビューの先頭**に大チャート（h:140）＋最新値・前年比＋注意書き:
    「※ドル換算のため為替変動（円安・ユーロ安）の影響を含む」。id `GLOBAL_LIQ_USD` を series から引いて表示。無ければカード非表示。

## 4. B2 金利差（汎用スプレッド）

- `fetch_spread(minuend_id, subtrahend_id, api_key, fred_fetcher=fetch_series) -> list`:
  - FRED 2系列を取得し、**minuendの日付に整列**（subtrahend は `_nearest_prior`）、`差 = minuend − subtrahend`（4桁丸め）。
- `computed_dispatch` に `spread` を追加（config の `minuend`/`subtrahend` キーを参照）。
- 設定3系列（indicator「金利差」, unit "%", transform "level", group "market"）:
  | id | country | minuend | subtrahend |
  |---|---|---|---|
  | SPREAD_UST10_JGB10 | 日米10年金利差 | DGS10 | IRLTLT01JPM156N |
  | SPREAD_UST10_BUND10 | 米欧10年金利差 | DGS10 | IRLTLT01DEM156N |
  | SPREAD_FF_ECB | 米欧政策金利差 | FEDFUNDS | ECBDFR |
- Zスコア/ムーバー等の既存シグナルが自動適用（risk_dir は付けない）。

## 5. D2 ブリーフィングに政策文脈

- `build_briefing(series, signals, as_of, events=None)` に拡張:
  - `events` は calendar のリスト（`{date, name_ja, category, ...}`）。`as_of <= date <= as_of+14日` を抽出し日付順で
    `今後の政策・会議予定: {MM/DD} {name}({category}) / ...`（最大6件）行を「データ鮮度」行の前に追加。該当なしなら行を出さない。
- `build_macro_payload(..., calendar=None)` にパラメータ追加し build_briefing へ渡す。
- `main()`: `data = load_data()` の後に `data["macro"] = build_macro_payload(calendar=data.get("calendar"))` と呼ぶ。

## 6. エラーハンドリング・正確性
- 構成系列の取得失敗はそのcomputed系列ごとスキップしログ（既存方針）。
- 為替換算の注意書きを表示（量と為替効果の混在を明示）。値はFRED公式のみ。
- events の日付は文字列比較（ISO形式）で安全に。

## 7. テスト
- `fetch_global_liq`: 5系列のダミーを注入し、単位換算・WALCL整列・(検算例: WALCL 6,735,645 / ECB 6,119,940×1.147 / BOJ 6,643,630×100÷161.37 → ≈17.87兆ドル) を検証。欠損日はスキップ。
- `fetch_spread`: 2系列の整列と差。
- `build_briefing`: events 14日以内の抽出・整形・0件時の非表示。
- payload/表示: GLOBAL_LIQ_USD がマネーの流れ先頭に出る（ブラウザ確認）。既存テスト全PASS。
