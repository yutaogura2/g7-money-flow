# シグナル/解釈レイヤー（サブA）設計書

- 作成日: 2026-06-22
- 対象: 既存「G7一次情報＋マクロ数値ダッシュボード」への拡張
- 位置づけ: プロ向け拡張（A→B→C）の **サブA**。B（データ拡充）・C（配信）は別サイクル。

## 1. 目的

並んだ数値を「**今どこが極端で、どんな局面か**」が分かるシグナルに変える。各指標にZスコア/パーセンタイル・信号色を付け、概要に「リスクレジーム」と「直近の主な変化」を出す。**新規データ取得はなし**（既存の全履歴CSVから計算）。

## 2. スコープ（確定要件）

- A1 Zスコア/パーセンタイル（**全期間**基準）
- A2 しきい値フラグ（`risk_dir`＋符号付きZ＝stress、信号色）
- A3 リスクレジーム総合（stress平均→ラベル）
- A4 直近の主な変化（ムーバー、直近1か月のΔz上位）
- 表示：各カードにZ/%ile・信号色／概要にレジーム・パネル＋ムーバー一覧

### スコープ外（YAGNI）
- 新規データ源（B）、自動ブリーフィング/チャート（C）、バックテスト。

## 3. 計算仕様（build.py）

### A1 統計
- `_series_stats(values: list[float]) -> dict`：**全期間**の値から
  - `mean`、`std`（母標準偏差, n）、
  - `zscore = (latest - mean)/std`（std==0 は 0、round 2桁）、
  - `pctile = (count(v <= latest)/N)*100`（round 0桁）。
- `latest` は系列末尾（最新）の値。

### A2 stress（リスク方向の符号付きZ）
- 系列に `risk_dir` があるとき：`stress = zscore if risk_dir=="high" else -zscore`（高いほどリスクオフ=正）。
- `risk_dir` 無しは `stress = null`。

### A3 regime
- `signals.regime.score = mean(stress for risk_dir系列)`（round 2桁）。
- ラベル: `score>=0.5`→「リスクオフ」、`score<=-0.5`→「リスクオン」、その間「中立」。
- `level`: 同様に warn/ok/calm（色分け用：リスクオフ=warn(赤系)、リスクオン=calm(緑系)、中立=ok）。

### A4 movers（直近1か月のΔz）
- 各系列で、最新日 `d0` と `d0` の約30日前（`_nearest_prior(history, d0-30d)`）の値から
  `delta = latest - prev`、`delta_z = delta/std`（std==0 は 0）。
- `signals.movers` = `|delta_z|` 降順 上位6件。各要素 `{id, country, indicator, latest, delta, delta_z, dir}`（dir は delta 符号 "up"/"down"）。

## 4. payload 追加（build_macro_payload）

各系列dictに `zscore`・`pctile`・`stress`（risk_dir無しはnull）を追加。
トップに `signals = { "regime": {score,label,level}, "movers": [...] }` を追加。
- 統計は **truncate前の全CSV値** から計算（表示用 `history` は従来どおり `history_points` 件に間引き）。

## 5. series_config（risk_dir 付与）

シグナル系列に `risk_dir` を追記（他は不変）:

## 6. index.html の変更

- `macroCard`：最新値の行の下に「Zスコア {z} ・ {pctile}%ile」を表示。`stress` があれば信号色ドット＋ラベル（`stress>=1.5`🔴強ストレス／`>=0.5`🟠やや／`<=-0.5`🟢緩和／その間 中立）を表示。
- `renderOverview`：先頭に
  - 「リスクレジーム」パネル（`signals.regime.label` を色付き大表示＋score）。
  - 「直近の主な変化」リスト（`signals.movers` を方向アイコン付きで6件）。
- 既存のチャート/レンジキャプション/カードは流用。マクロ・市場リスク両ビューに自動適用（macroCard共通）。

## 7. エラーハンドリング・正確性
- 履歴が極端に短い/std==0 の系列はZ=0・stress=0で安全に扱う（除算回避）。
- `signals.regime` は risk_dir 系列が1つも無い場合は score=null・label「—」。
- 値は既存CSV（公式データ）由来のみ。新規取得・推測なし。

## 8. テスト
- `_series_stats`: 既知値（例 [1,2,3,4,5]、latest=5）→ mean=3, std≈1.414, z≈1.41, pctile=100 を検証。std==0 で z=0。
- stress 符号: risk_dir=high で +z、low で −z。
- regime: 複数 stress の平均とラベル境界（0.5/−0.5）。
- movers: Δz 計算と上位順、dir 符号。
- payload に zscore/pctile/stress と signals が入ること。既存テスト全PASS。
